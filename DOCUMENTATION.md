## 1. 系统概述

### 1.1 系统定位

**麻精药品智能柜售后运维工具 v2.0** 是针对麻精药品智能柜全生命周期管理的专业售后运维系统，旨在提升设备运维效率、规范服务流程、沉淀知识经验。

### 1.2 核心价值

| 维度 | 价值描述 |
|------|---------|
| 📋 **流程标准化** | 工单五阶闭环管理：报修→排查→修复→复核→回访 |
| 📊 **数据可视化** | 设备状态、故障趋势、响应时长实时监控 |
| 🧠 **知识沉淀** | 故障解决方案自动入库，形成可检索知识库 |
| 🔐 **权限管控** | 三级角色体系，数据隔离，审计留痕 |
| 📱 **轻量部署** | 零配置开箱即用，支持单机/局域网/云部署 |

### 1.3 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                      前端层 (SPA)                          │
│  web/index.html + web/css/style.css + web/js/app.js        │
├─────────────────────────────────────────────────────────────┤
│                      后端层 (Node.js)                      │
│  server/server.js + server/db.js + server/auth.js          │
├─────────────────────────────────────────────────────────────┤
│                      数据层 (SQLite)                       │
│  data/yt_asmt.db                                          │
└─────────────────────────────────────────────────────────────┘
```

## 4. 功能模块详解

### 4.1 工作台

**功能定位**：全局概览，快速入口

**核心功能**：
- 📊 故障统计概览（今日新增/待处理/已闭环）
- 📦 设备状态概览（在线/离线/异常）
- 🔔 待办事项提醒
- ⚡ 快捷操作入口

### 4.2 工单管理

**功能定位**：故障工单全生命周期管理

**五阶流程**：
```
报修登记 → 现场排查 → 故障修复 → 复核闭环 → 售后回访
   ↓           ↓          ↓           ↓          ↓
  录入信息   诊断问题    执行修复     总部确认    客户反馈
```

**工单状态**：
| 状态 | 说明 | 操作权限 |
|------|------|---------|
| 待处理 | 新建工单，等待处理 | 工程师/经销商 |
| 处理中 | 工程师正在现场排查 | 工程师 |
| 待复核 | 修复完成，等待总部复核 | 总部 |
| 已闭环 | 复核通过，工单完成 | 全部角色 |
| 已回访 | 售后回访完成 | 全部角色 |

### 4.3 设备管理

**功能定位**：设备全生命周期管理

**核心功能**：
- 📱 设备绑定（一机一码）
- 📋 设备状态变更
- 📊 设备详情查看
- 🔗 关联工单/巡检记录

**设备状态**：在线、离线、异常、维护中、已停用

### 4.4 巡检管理

**功能定位**：定期巡检计划与执行

**核心功能**：
- 📅 巡检计划创建与管理
- ✅ 巡检记录执行与提交
- 📊 巡检统计分析

**巡检项目**：
- 外观检查、安装检查、固件版本、网络状态
- 药品库存、硬件状态、温湿度监测

### 4.5 维保台账

**功能定位**：设备维护保养记录

**维保类型**：
| 类型 | 说明 |
|------|------|
| 配件更换 | 更换设备配件 |
| 固件升级 | 升级设备固件 |
| 清洁保养 | 设备清洁保养 |
| 校准检测 | 设备校准检测 |

### 4.6 需求管理

**功能定位**：收集和管理用户需求

**流程**：需求提交 → 总部评估 → 采纳/驳回

**需求状态**：待评估、已采纳、已驳回、开发中、已完成

### 4.7 知识库

**功能定位**：故障解决方案沉淀与检索

**核心功能**：
- 🔍 全文搜索
- 📝 知识条目管理
- 🔗 工单关联自动入库

### 4.8 数据统计

**功能定位**：运维数据可视化分析（仅总部可见）

**统计维度**：
- 📊 故障分布与趋势
- 📈 设备在线率
- ⏱️ 平均响应时长
- 👤 工程师绩效

### 4.9 审计日志

**功能定位**：操作记录追溯（仅总部可见）

**记录内容**：
- 用户操作时间、操作人、操作类型
- 操作对象、变更内容

## 5. 操作指南

### 5.1 工单操作流程

#### 步骤1：创建工单
```
1. 点击左侧菜单「工单管理」
2. 点击「新建工单」按钮
3. 填写工单信息：
   - 设备编号（必填）
   - 故障类型（必填）
   - 故障描述（必填）
   - 优先级（默认一般）
4. 点击「提交」
```

#### 步骤2：处理工单
```
1. 在工单列表中找到待处理工单
2. 点击工单查看详情
3. 填写「现场排查」信息
4. 选择「处理中」状态
5. 提交排查结果
```

#### 步骤3：完成修复
```
1. 在工单详情页
2. 填写「修复方案」
3. 上传修复照片（可选）
4. 选择「待复核」状态
5. 提交修复结果
```

#### 步骤4：复核闭环（总部）
```
1. 在工单列表中筛选「待复核」工单
2. 查看修复详情
3. 确认修复结果
4. 选择「已闭环」状态
5. 完成复核
```

### 5.2 设备绑定流程

```
1. 点击左侧菜单「设备管理」
2. 点击「设备绑定」按钮
3. 扫描设备二维码或手动输入设备编号
4. 选择医院/科室
5. 填写设备型号、安装位置等信息
6. 点击「绑定」完成
```

### 5.3 巡检执行流程

```
1. 点击左侧菜单「巡检管理」
2. 找到需要执行的巡检计划
3. 点击「执行巡检」按钮
4. 填写巡检记录：
   - 外观检查（是/否正常）
   - 网络状态（是/否稳定）
   - 硬件检测结果
   - 巡检照片（可选）
5. 选择巡检结果（正常/异常待处理/已处理）
6. 点击「提交」
```

## 6. 部署指南

### 6.1 单机部署

```bash
# 1. 安装 Node.js v16+
# 2. 解压项目到 E:\YT-ASMT
# 3. 双击 start.bat
# 4. 访问 http://localhost:3000
```

### 6.2 局域网部署

```bash
# 1. 开放防火墙端口
netsh advfirewall firewall add rule name="YT-ASMT Web" dir=in action=allow protocol=TCP localport=3000

# 2. 启动服务
cd E:\YT-ASMT\server
npm install
node server.js

# 3. 局域网访问
# http://[服务器IP]:3000
```

### 6.3 云服务器部署（生产环境）

#### 6.3.1 环境准备
```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Node.js（使用 nvm）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
source ~/.bashrc
nvm install 20.17.0
nvm alias default 20.17.0
```

#### 6.3.2 部署项目
```bash
# 创建项目目录
mkdir -p /opt/YT-ASMT
cd /opt/YT-ASMT

# 克隆代码（替换为你的仓库地址）
git clone https://github.com/JamesMeredith/YT-ASMT.git .

# 安装依赖
cd server
npm install --production
```

#### 6.3.3 进程管理（PM2）
```bash
# 安装 PM2
npm install -g pm2

# 启动服务
pm2 start server.js --name yt-asmt

# 设置开机自启
pm2 save
pm2 startup
```

#### 6.3.4 Nginx 反向代理（推荐）
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /path/to/ssl/certificate.crt;
    ssl_certificate_key /path/to/ssl/private.key;
    
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 7. 故障排除

### 7.1 常见问题

#### 问题1：端口被占用
**现象**：启动时报错 `EADDRINUSE: address already in use`

**解决方案**：
```bash
# 查找占用进程
netstat -ano | findstr ":3000"

# 终止进程（替换 PID 为实际进程号）
taskkill /f /pid <PID>

# 或换端口启动
set PORT=8080
node server.js
```

#### 问题2：登录失败（无反应）
**现象**：点击登录按钮无反应，控制台报错 `doLogin is not defined`

**解决方案**：
```bash
# 检查脚本加载
# 1. 按 F12 打开开发者工具
# 2. 查看 Console 标签是否有错误
# 3. 检查 Network 标签确认脚本加载正常
# 4. 强制刷新页面（Ctrl+Shift+R）
```

#### 问题3：数据库文件权限问题
**现象**：启动时报错 `SQLITE_READONLY`

**解决方案**：
```bash
# Windows
icacls data /grant Everyone:F /T

# Linux
chown -R www-data:www-data /opt/YT-ASMT/data
chmod -R 755 /opt/YT-ASMT/data
```

#### 问题4：中文乱码
**现象**：日志或页面显示乱码

**解决方案**：
```bash
# 确保文件编码为 UTF-8
# 在 IDE 中检查文件编码设置
# 确保数据库连接使用 UTF-8
```

#### 问题5：无法访问（局域网）
**现象**：本机可访问，其他电脑无法访问

**解决方案**：
```bash
# 检查防火墙
netsh advfirewall firewall show rule name="YT-ASMT Web"

# 确认端口监听
netstat -ano | findstr "LISTENING" | findstr ":3000"

# 测试网络连通性
ping [服务器IP]
telnet [服务器IP] 3000
```

### 7.2 日志查看

```bash
# Windows（启动窗口直接查看）
# 日志会实时输出到控制台

# Linux（PM2 日志）
pm2 logs yt-asmt

pm2 logs yt-asmt --lines 50
```
