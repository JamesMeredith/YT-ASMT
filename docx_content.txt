1.2 设备安装规范
柜体离墙距离 ≤ 10cm，安装表单提交时强制校验
立式设备需额外校验地面水平度（倾斜角 ≤ 2°）

安装完成后自动生成安装报告 PDF，院方签字确认
1.3 工具初始化规则
启动时自动检测网络状态：无外网 → 开启离线模式；有外网 → 在线模式 + 尝试同步离线队列
自动开启合规审计日志：全操作留痕，采用追加写入模式，不可篡改、不可删除
根据系统时间（6:00-18:00）自动匹配深色/浅色主题，适配医院强光环境，支持手动切换
自动加载当前登录人信息（姓名、角色、所属省份/区域），并缓存至本地
首次启动强制引导完成「设备绑定」或「加入已有设备组」，否则禁止进入主界面
自动检测本地数据库版本，如需迁移则自动执行升级脚本
1.4 通用工具规则
日期格式全局统一为：YYYY-MM-DD HH:mm（24小时制），存储与展示均使用北京时间
敏感信息自动脱敏：手机号显示 138****1234，IP 地址末段脱敏，内网地址完全隐藏
附件自动命名规则：[业务节点]_[医院简称]_[YYYYMMDDHHmmss].[扩展名]
列表默认按创建时间降序排列，每页 20 条，支持上拉加载更多
全局异常捕获：未捕获异常弹出友好提示并自动记录错误日志，不崩溃退出
二、核心业务流程
售后两大核心主流程，严格遵循 PPT 标准规范，覆盖故障处理与需求管理全生命周期。

2.1.1 流程总览
整体流程：报修登记 → 自动分级 → 现场排查 → 故障修复 → 复核闭环 → 售后回访
每个节点强制顺序流转，不可跳步，当前节点完成后自动解锁下一节点
全流程时间轴可视化展示，每个节点记录操作人、操作时间、操作内容

2.1.4 现场排查
强制上传现场排查照片（至少 1 张含设备全景），无照片禁止提交
排查结果字段：现象确认（已复现/未复现/间歇性）、根因分析（文本，必填）、排查时间（自动记录）
若排查结论为「非设备故障」（如操作培训问题），可选择直接闭环并记录培训完成情况
2.1.5 故障修复
修复方案：文本必填，至少 20 字
强制上传修复完成照片（至少 1 张，含修复后设备状态），无照片禁止提交
如涉及配件更换，自动关联维保台账，记录更换配件型号、数量
提交后自动更新故障状态为「处理中」→「待复核」
支持记录备件消耗，库存不足时自动提醒补货
2.1.6 复核闭环
复核结果：正常 / 异常（异常时回退至修复环节）
仅当复核结果为「正常」时，允许闭环。闭环确认 = 二次弹窗确认，防止误操作
闭环时自动执行：
将故障类型+解决方案结构化存入知识库
更新设备状态为「正常」
记录复核人、复核时间、院方确认方式（签字/拍照）
触发生成本次故障处理报告（PDF），可分享给院方
2.1.7 售后回访
回访方式：电话/现场/微信
评分项：服务响应速度、故障解决质量、工程师专业度（1-5 星）
满意度：非常满意/满意/一般/不满意（不满意时强制填写原因）
回访记录自动归档，关联对应故障单号，纳入工程师绩效考核
闭环后 7 天内未完成回访 → 自动提醒对应工程师
2.2 新功能/改进需求处理流程
2.2.1 流程总览
整体流程：需求登记 → 总部评估（采纳/驳回） → 进度同步 → 归档
全程可追溯，每个状态变更自动通知需求提交方
2.2.2 需求登记
自动生成唯一需求编号：XQ_[YYYYMMDD]_[3位序号]
必填：需求标题、需求描述（≥ 20字）、需求来源（医院名称自动关联）、优先级建议
可选：附件截图/示意图（jpg/png，最多 3 张）
2.2.3 需求评估（仅总部权限）
评估结论：已采纳（填写预计上线时间、排期说明）/ 已驳回（填写驳回原因，≥ 10字）
评估后自动推送通知至需求提交方的医院联系人
支持评估备注（内部可见，不推送给医院）
2.2.4 进度跟踪与归档
已采纳需求：状态流转 → 开发中 → 测试中 → 已上线，每个状态变更自动推送通知
已驳回需求：留存备案，支持院方补充材料后重新提交
所有需求变更记录形成操作日志，可追溯每次状态变更的时间与操作人
三、增值运维功能
主动式售后管理功能，从被动响应转向预防性维护，实现设备全生命周期管理。
3.1 定期巡检
巡检计划创建：选择医院 → 选择设备（可多选）→ 设置巡检周期（每周/每两周/每月）→ 设置开始日期
巡检待办自动生成：到达巡检日期后，自动生成待办任务并推送负责工程师
巡检检查项：
设备外观与安装（柜体完整性、离墙距离、水平度）
软件状态（系统版本号、应用版本号、运行时长）
网络状态（IP 地址、连接稳定性、丢包率）
药品库存预警（低于安全库存数量、即将过期药品）
硬件检查（屏幕、扫描器、打印机、锁控）
巡检完成自动生成巡检报告并归档；异常项自动标记为「异常待处理」，关联创建故障工单
3.2 维保台账
自动生成维保编号：WB_[设备编码后6位]_[YYYYMMDD]_[序号]
维保类型：配件更换 / 固件升级 / 清洁保养 / 校准调试 / 其他
配件更换时自动记录：配件名称、型号、数量、批次号
维保记录与设备唯一编码终身绑定，形成完整维保履历
支持按设备/时间段/维保类型多维度查询与导出
3.3 设备台账绑定（一机一码）
扫码/手动输入后查重，已存在则提示"该编码已绑定至XX医院"，禁止重复
绑定信息：设备型号（台式/立式）、序列号、安装时间、安装位置（楼层/科室）、网络 IP
离墙距离自动校验：安装表单提交时比对标准值，超标弹出红色警告并禁止提交
支持设备状态变更：在线/离线/维修中/已报废，状态变更记录操作日志
四、QClaw 平台专属能力
4.1 离线数据同步
离线模式判定：连续 3 次 Ping 云端服务器失败 → 自动切换离线模式，界面顶部显示黄色离线标识
离线操作范围：所有表单提交、流程流转、附件上传均在本地 SQLite 缓存
同步策略：
自动同步：检测到网络恢复后，每 30 秒执行一次增量同步（逐条上传未同步数据）
手动同步：工程师可随时点击「一键同步」强制执行全量同步
冲突策略：以本地数据为准（本地时间戳 > 云端时间戳），冲突数据标记待人工审核
同步进度：界面底部显示同步队列数量与进度条
同步失败重试：单条失败自动重试 3 次，间隔 5s/15s/30s；3 次均失败则跳过并标记异常，同步完成后汇总提示

4.3 合规审计
审计范围：登录/登出、表单提交/修改/删除、流程节点变更、权限变更、数据导出
日志字段：操作时间、操作人、角色、IP 地址、操作类型、操作对象、变更前值、变更后值
日志存储：本地数据库独立表，采用追加模式（Append-Only），物理删除禁止
仅总部角色可查看审计日志并导出为 Excel，导出操作本身也记入审计日志
4.4 消息推送
推送渠道：工具内通知中心 + 短信（重大故障）+ 企微/邮件（总部）
推送节点：
重大故障 → 总部技术部 + 区域负责人（短信+通知）
紧急故障 → 区域负责人（通知）
需求评估完成 → 医院联系人（通知）
流程节点流转 → 下一节点处理人（通知）
巡检待办到期 → 负责工程师（通知）
消息已读/未读状态追踪，未读消息红点角标提示

六、知识库与数据统计
6.1 知识库
入库规则：故障闭环后，自动提取「故障分类 + 故障描述摘要 + 解决方案 + 涉及设备型号」结构化入库
支持手动新增知识条目（总部权限）
检索方式：关键词全文搜索 + 故障分类筛选 + 设备型号筛选
知识条目展示：标题、解决方案摘要、适用型号、引用次数、更新时间
使用统计：记录每次知识库检索与引用，引用越多排越前
注意：知识库是提高故障处理效率的关键，鼓励工程师优先检索知识库再现场排查。

九、异常与边界处理规范
9.1 网络异常
断网：自动切换离线模式，界面顶部黄色横幅提示，所有操作本地缓存
弱网（请求超时 > 5s）：提示"当前网络较慢"，允许继续操作本地功能
同步冲突：同时操作同一工单 → 以后提交者为准，先提交者数据标记为"覆盖"并通知
9.2 数据异常
设备编码重复绑定：禁止提交，提示已有绑定信息
数据库损坏：自动尝试修复，失败则提示"建议重新安装"并导出所有可读数据
附件损坏/丢失：在附件位置显示"文件已失效"，不影响其他数据显示
9.3 操作异常
重复提交：按钮点击后立即禁用（300ms 防抖），后端基于单号幂等校验
流程回退：仅总部可手动回退流程至上一节点（需填写回退原因并计入审计日志）
工单误闭环：闭环后 24 小时内支持总部「撤销闭环」操作
十、部署与执行规则
10.1 部署流程
步骤 1：将本开发脚本完整导入 QClaw 小龙平台
步骤 2：QClaw 自动解析脚本规则，生成数据表结构、业务逻辑、流程引擎配置
步骤 3：平台自动生成 PC 客户端 + 安卓平板安装包
步骤 4：总部管理员登录，创建经销商/工程师账号，分配角色权限
步骤 5：工程师下载安装包，首次登录完成设备绑定
步骤 6：离线安装包可拷贝至 U 盘，在医院现场直接安装运行，无需网络配置
10.2 版本升级
工具版本号格式：主版本.次版本.修订号（如 2.0.1）
升级检测：联网时自动检测最新版本，提示用户升级
强制升级：重大安全修复版本标记为强制升级，旧版本禁止登录
